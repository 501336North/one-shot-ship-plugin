# One Shot Ship - Development Workflows

> **Your unfair advantage for shipping software.**

This project uses One Shot Ship workflows for professional development standards.

## Available Commands

### Development Voyage
- `/oss:read-the-winds "idea"` - Brainstorm and refine ideas
- `/oss:plot-the-course "feature"` - Create implementation plans
- `/oss:set-sail` - Execute the plan
- `/oss:dock-ship` - Quality check, commit, PR, merge

### Sea Trials (Testing)
- `/oss:test-waters` - Run E2E tests
- `/oss:measure-tempest` - Performance benchmarks
- `/oss:scan-pirates` - Security scanning

### Deployment
- `/oss:ship-harbor` - Deploy to staging
- `/oss:sail-horizon` - Deploy to production
- `/oss:hoist-colors` - Production with fanfare

### Operations
- `/oss:lookout-watch` - Monitor production
- `/oss:sound-alarm` - Incident response
- `/oss:return-to-port` - Emergency rollback

## Development Standards

This project enforces:
- **Quality Gates**: Pre-commit and pre-push hooks
- **Security**: Automated vulnerability scanning
- **Performance**: Benchmark tracking
