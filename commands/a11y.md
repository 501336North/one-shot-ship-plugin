# /oss:a11y - Accessibility Audit

WCAG 2.2 accessibility audit and compliance checking.

## What This Command Does

1. **WCAG scan** - Check against WCAG 2.2 AA/AAA criteria
2. **ARIA validation** - Verify proper ARIA usage
3. **Color contrast** - Check color contrast ratios
4. **Keyboard navigation** - Test keyboard accessibility
5. **Screen reader** - Verify screen reader compatibility

## Step 1: Check Authentication

```bash
cat ~/.oss/config.json 2>/dev/null | grep apiKey
```

If no API key found:
```
No API key found. Run: /oss:login
Register at https://www.oneshotship.com
```

## Step 2: Initialize Logging

**You MUST initialize logging for supervisor visibility.**

```bash
$CLAUDE_PLUGIN_ROOT/hooks/oss-log.sh init a11y
```

## Step 3: Fetch Prompt from API

```
URL: https://one-shot-ship-api.onrender.com/api/v1/prompts/workflows/a11y
Method: GET
Headers:
  Authorization: Bearer {apiKey}
```

## Step 4: Execute the Fetched Prompt

The prompt handles:
- WCAG 2.2 compliance checking
- Automated accessibility testing
- Manual audit guidance
- Remediation recommendations

## Example Usage

```bash
/oss:a11y
/oss:a11y --level AAA
/oss:a11y src/components/
```
